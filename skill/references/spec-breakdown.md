# Break a Spec into Sprints

Use when translating a spec into sprint/chunk plans.

## Planning requirements

1. Keep execution sequential and dependency-safe.
2. Ensure each planned chunk outputs clear artifacts and verifiable criteria.
3. Include test expectations early; do not defer all validation to the end.
4. Assume post-sprint automation will run review/docs/tests, and plan artifacts accordingly.
5. Treat `SCRATCHPAD.md` as required for every sprint to preserve negative knowledge across context resets.
